  • git 版本规划

    • 主分支 master
      • 功能开发分支 feature/{功能描述}
        • 测试分支 test
          • 发布分支 release
            • 临时 bug 修复 hotfix

            # git 版本规划

            # 主分支 master

            与线上版本保持一致，随时可以发布 不做任何 commit 接受来自 release，hotfix 的 merge

            # 功能开发分支 feature/{功能描述}

            具体功能、特性的工作分支族，大部分的 commit 在此类分支上完成。

            # 测试分支 test

            主要用于接受功能分支的合并 前端构建，并且打上测试 tag

            # 发布分支 release

            上线前的 release 分支，接受需要上线的功能分支的合并,同时删除具体的 feature 分支 如果有 bug，直接在当前修复 完成测试和开发后，构建完，打完 tag 合并到 master 分支

            # 临时 bug 修复 hotfix

            用于在 master 分支发现实际环境中问题进行修复，接收修复 commit。 完成后合并到 release 分支，打 tag,最终合并到 master 分支

            tag: 170.0.0.1 采用 4 位方式，一般上线为 170.1.0.0，普通的提测为 170.0.0.2
